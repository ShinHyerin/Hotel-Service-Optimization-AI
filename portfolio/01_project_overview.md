# 📄 01. 프로젝트 기획서 (Project Overview)

## 1. 기획 배경 및 의도 (Project Background & Intent)

현대 호스피탈리티(호텔) 및 자원 공유 공간(공유오피스) 산업은 예약부터 실제 이용, 그리고 사후 응대에 이르기까지 다양한 단계에서 자원 낭비와 비효율성을 초래하고 있습니다. 

본 통합 프로젝트는 **정형 데이터 예측, 컴퓨터 비전(Computer Vision), 자연어 처리(NLP)** 기술을 하나의 파이프라인으로 유기적으로 엮어, **"예약 현황 예측 ➔ 현장 공간 운영 ➔ 고객 및 직원 응대"**의 전 과정을 AI 기반으로 최적화하는 **스마트 호스피탈리티 통합 솔루션**을 제안합니다.

---

## 2. 핵심 서비스별 상세 기획

### 🏨 [예약/모니터링] Flask 기반 실시간 객실 취소 발생 확률 예측 시스템 (B2C)
* **대상 사용자** : 만실인 인기 호텔의 공실(취소표)을 확보하고자 하는 일반 고객
* **기획 의도** : 
  예약이 가득 찬 상황에서 사용자가 특정 조건(리드 타임, 국가 등)을 입력하면 AI가 실시간 취소 발생 가능성(공실 전환율)을 예측합니다. 사용자는 무작정 새로고침을 하는 대신, 티켓팅 취소표를 저격하듯 데이터 기반으로 취소 확률이 높은 시점과 조건을 파악하여 영리하게 예약을 시도할 수 있는 모니터링 환경을 제공합니다.

### 🖥️ [현장 운영] YOLOv11 기반 실시간 공간 이용 실태 분석 시스템 (B2B)
* **대상 사용자** : 세미나실 및 공유오피스 공간 관리자
* **기획 의도** : 
  예약 완료 이후 발생하는 세미나실의 노쇼(No-Show), 장시간 자리비움, 미정리 퇴실 등의 현장 비효율을 **Object Detection(객체 탐지)** 기술로 실시간 탐지합니다. 이를 통해 물리적 공실 회전율을 극대화하고, 인력 중심 관리의 한계를 정밀한 시각 데이터로 극복합니다.

### 💬 [서비스 응대] Hugging Face SLM 기반 가상 호텔 사내 규정 안내 챗봇
* **대상 사용자** : 외부 투숙객 및 호텔 내부 신입 직원
* **기획 의도** : 
  NER(개체명 인식) 및 QnA 파이프라인을 결합한 소형 언어 모델(SLM)을 구축하여 언어적 모호함을 제거합니다. 투숙객에게는 24시간 즉각적인 규정(조식, 시설 이용 등)을 안내하고, 신입 직원에게는 복잡한 매뉴얼 검색을 자동화하여 휴먼 에러를 방지하고 서비스 신뢰도를 상향 평준화합니다.

## 3. 개발 목표 (Project Goals)

### 1️⃣ [ML/Flask] 실시간 취소 확률 예측 및 비동기 예약 데이터 축적 시스템
* **Kaggle 기반 도메인 데이터셋 전처리 및 적재**
  * Kaggle의 *'Hotel booking demand'* 데이터셋을 정제하여 Oracle 11g DB에 안정적으로 연동 및 적재 (`db_config.py`, `upload_to_oracle.py`).
* **랜덤 포레스트 기반 모델 최적화 및 Feature 엔지니어링**
  * `is_canceled`를 타깃 변수로 설정하고, 변수 중요도(Feature Importance) 분석을 통해 불필요한 변수를 제거하여 예측 모델의 성능과 효율성 극대화 (`train.py`).
* **완전 비동기(Fetch API) 예측 대시보드 및 실시간 피드백 구현**
  * 사용자의 8가지 핵심 조건(리드 타임, 국적 등) 변경 시, 화면 새로고침 없이 실시간으로 AI 취소 예측 확률을 반영하는 모던 프론트엔드 구축 (`predict.html`).
* **사용자 행동(취소) 기반 예측 가중치 갱신 파이프라인 구축**
  * 예약 조회 후 **[예약 취소]** 수행 시, 해당 고객의 과거 취소 이력(`previous_cancellations`)을 즉시 가산(+1)하고 백엔드 모델에 실시간 반영하여 데이터가 축적되는 선순환 흐름 검증.

---

### 2️⃣ [Computer Vision/YOLOv11] 실시간 공간 이용 실태 및 노쇼 판별 시스템
* **YOLOv11 기반 멀티 클래스 고속 객체 탐지**
  * 실시간 CCTV 영상 프레임 내에서 사람(`Person`), 노트북(`Laptop`), 가방(`Bag`) 등의 다중 객체를 1초 이내($Latency < 1s$)로 동시 탐지.
* **관계 인지형 비즈니스 로직(Conditional Logic) 설계**
  * **노쇼(No-Show):** 예약 시간 내 사람 및 소지품이 모두 탐지되지 않는 상태 판별.
  * **자리비움:** 사람은 없으나 노트북/가방이 남아있는 일시적 부재 상태를 인지하여 오탐 패널티 제외.
  * **이용률 저조:** 예약 인원 대비 실제 입실 인원이 부족한 상태 판별.
* **RESTful API 기반 확장형 아키텍처 아웃라인**
  * 프론트엔드와 백엔드 간 추론 결과 데이터를 JSON 포맷으로 통신하여, 향후 여러 개의 세미나실 공간(Room ID)이 확장되더라도 유연하게 대응할 수 있는 확장성 확보.

---

### 3️⃣ [NLP/SLM] 매뉴얼 기반 가상 호텔 사내 규정 안내 챗봇 (Hugging Face)
* **SLM 기반 고효율·저비용 추론 환경 구현**
  * 고가의 GPU 인프라 없이 단일 GPU 혹은 로컬 개발 환경에서 3초 이내에 답변 출력이 가능한 경량 언어 모델(SLM) 구축.
* **자연어 쿼리 처리 및 의도 파악(NER) 고도화**
  * 사용자의 정제되지 않은 구어체나 대화체 표현 속에서 의도와 핵심 엔티티(장소, 물품 등)를 정확히 추출하는 파이프라인 구성.
* **환각 현상(Hallucination) 제어 및 팩트 기반 답변 체계 확립**
  * 사내 운영 매뉴얼 문서 범위 내에서만 답변하는 QnA 시스템을 구축하고, 범위를 벗어난 질의에 대해서는 지정된 방어형 답변을 출력하는 안전장치 마련.
* **데이터 독립성 및 보안성 확보**
  * 임베딩 및 프롬프트 주입 단계에서 외부 Open API를 거치지 않고 완전히 로컬(On-Premise) 환경 내에서 처리하여 호텔 내부 보안 규정 준수.
