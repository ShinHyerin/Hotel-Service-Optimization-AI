## 기능 ID: F-02-01

- **기능명:** 실시간 AI 예측 및 DB 누적 통계 연동형 예약 시스템 (비동기)

### 1. 화면 및 UI 구성

- **상단:** 예약 자원 선택 (라디오 버튼: `Resort Hotel` / `City Hotel(세미나실)`).
- **중앙:** 고객 정보 입력 폼 및 AI 예측 피처용 체크리스트 (날짜, 고객 유형, 국가, 요청 사항 등).
- **하단:** * **시각화 영역:** 두 개의 독립된 차트 객체 (`resortChart`, `cityChart`) 또는 가변형 단일 차트 레이아웃.
    - **액션 버튼:** [예약저장] 버튼.

### 2. 시나리오별 프론트엔드 액션 & 비동기 통신 (Fetch API)

### ① 페이지 최초 진입 및 타입 미선택 상태

- 두 그래프(`resortChart`, `cityChart`)는 모두 데이터가 없는 빈 상태(또는 0%)로 렌더링되거나 숨김 처리됨.

### ② 사용자가 특정 타입 선택 (예: 세미나실 / City Hotel)

- **액션:** 라디오 버튼에서 `City Hotel`을 클릭하는 순간 트리거 발생.
- **비동기 요청 (Fetch 1):** 즉시 `fetch('/api/chart-data')`를 호출하여 현재까지 **DB에 축적된 전체 누적 평균 취소율 데이터**를 가져옴.
- **화면 반영:** * 반대쪽 그래프(`resortChart`)는 데이터를 완전히 비우거나 비활성화함.
    - 선택된 `cityChart` 영역에만 DB 기반 누적 평균 취소율(예: 67%)을 기본 베이스라인으로 먼저 그려줌.

### ③ 사용자가 입력 값(체크리스트)을 변경할 때 (실시간 예측 단계)

- **액션:** 사용자가 날짜, 국가, 요청사항 등을 변경할 때마다 이벤트 발생 (`updatePrediction()`).
- **비동기 요청 (Fetch 2):** `fetch('/api/predict', { method: 'POST', body: ... })`를 통해 현재 입력 폼 데이터를 Flask로 전송.
- **화면 반영:** 페이지 새로고침 없이, AI 모델이 리턴한 **'현재 사용자 조건의 예상 취소율(예: 82%)'** 데이터로 `cityChart`를 실시간 업데이트(Overwrite)함.

### ④ [예약저장] 버튼 클릭 시 (데이터 축적 단계)

- **액션:** 사용자가 [예약저장] 버튼을 클릭함.
- **비동기 요청 (Fetch 3):** `fetch('/api/reserve', { method: 'POST', body: ... })`를 통해 최종 예약 정보와 AI가 예측했던 취소율 데이터를 백엔드로 전송.
- **화면 반영:** * 페이지를 벗어나지 않고, "예약이 완료되었습니다"라는 안내와 함께 **현재 작성된 `cityChart`의 그래프 상태(최종 예측 결과값)를 화면에 고정(Lock)** 시킴.
    - 내부적으로 DB에 행(Row)이 추가되었으므로, 시스템 전체의 누적 통계 베이스라인이 미세하게 갱신됨.

### ⑤ 페이지를 나가지 않고 다른 타입(Resort Hotel)으로 변경 시

- 다시 ②번 단계로 돌아감. `cityChart`는 즉시 비워지고, `resortChart`가 활성화되면서 새로운 데이터 축적/예측 사이클이 새로고침 없이(비동기로) 작동함.

### 3. 백엔드 비즈니스 로직 (Flask) 및 API 엔드포인트 설계

### [/api/chart-data] (GET) - 누적 통계 조회 API (★신규 추가)

1. 프론트엔드로부터 요청을 받으면 Oracle DB에 접속.
2. `SELECT` 통계 쿼리를 실행하여 `Resort Hotel`과 `City Hotel`의 전체 누적 평균 취소율을 각각 계산.
3. 계산된 static 통계 지표를 JSON 형태로 반환.JSON
    
    ```
    { "resort_avg": 38, "city_avg": 67 }
    ```
    

### [/api/predict] (POST) - AI 실시간 예측 API

1. 사용자가 현재 입력한 체크리스트 데이터를 수신.
2. 데이터를 머신러닝 모델 입력 포맷으로 가공 후 예측 확률(`predict_proba`) 계산.
3. 해당 조건의 실시간 예측 취소율 값을 JSON으로 반환.

### [/api/reserve] (POST) - DB 저장 API

1. 최종 예약 정보 및 계산된 취소율 데이터를 수신.
2. Oracle DB의 `RESERVATION` 테이블에 `INSERT` 수행하여 데이터를 영구 축적함.
3. 저장 성공 메시지(`status: "success"`)를 반환하여 프론트엔드가 그래프를 고정할 수 있도록 함.

### 4. 데이터베이스 변화 (Oracle 11g)

- **`/api/reserve` 호출 시:** `RESERVATION` 테이블에 새로운 예약 행이 생성되며, 예측된 취소율 데이터가 컬럼에 함께 누적 저장됨.
- 이로 인해 다음번 `/api/chart-data`가 호출될 때 평균 취소율 분모/분자가 변하여 **그래프에 실시간으로 데이터가 축적되는 효과**가 완벽하게 구현됨.

### 🚀 다음은 무엇을 할까요?

요청하신 "새로고침 없이 실시간으로 그래프가 바뀌고 DB에 축적되는 비동기 메커니즘"을 명세서에 완벽히 반영했습니다. 이제 AI와 코드를 짤 때 이 명세서를 보여주면 헤매지 않고 깔끔하게 짜줄 것입니다.

요구사항분석서와 기능명세서가 아주 훌륭하게 완성되었으니, 이제 다음 단계로 넘어가 볼까요?

1. **DB 테이블 설계:** Oracle 11g에 매핑할 `RESERVATION` 테이블 컬럼 구조와 DDL 스크립트 작성하기
2. **Flask 프로젝트 구조 및 파일 뼈대 잡기:** `app.py`와 비동기 JavaScript 통신을 위한 기본 틀 잡기

## 3. 예약내역검색 화면 (Search Page)

### 기능 ID: F-03-01

- **기능명:** 비동기 예약 내역 조회 및 실시간 시각화
- **화면 및 UI 구성:**
    - 상단: 고객 정보 검색창 (이름 또는 연락처 입력 필드) 및 [검색] 버튼.
    - 하단 컨텐츠 영역 (초기 상태는 숨김 또는 빈 상태):
        - **조회 성공 시:** 고객의 예약 리스트 테이블 및 해당 예약의 **취소율 데이터 시각화 컴포넌트(차트/인디케이터)** 노출.
        - **취소율 시각화 레이아웃:** 각 예약 항목별 혹은 누적 취소율 지표를 시각적으로 표시.
- **프론트엔드 액션 (Fetch API):**
    - 페이지 로드 시, 이전 '예약하기' 페이지에서 넘어온 정보가 스토리지에 있다면 검색창에 자동으로 입력하고 검색 함수 자동 실행.
    - [검색] 버튼 클릭 시, 검색 키워드를 담아 `Fetch API`를 통해 `/api/search` 엔드포인트로 **POST 또는 GET** 요청 전송.
    - **결과 처리:**
        - **데이터가 있을 경우:** 하단 영역의 제한을 풀고(DOM 조작), 받아온 예약 리스트와 취소율 데이터를 매핑하여 화면에 즉시 렌더링 (화면 새로고침 없음). Chart.js 등을 활용해 시각화 그래프 갱신.
        - **데이터가 없을 경우:** `alert("일치하는 고객이 없습니다.");` 경고창을 띄우고 하단 영역을 초기화.
- **백엔드(Flask) 로직:**
    - `/api/search` 엔드포인트에서 검색 키워드 수신.
    - Oracle DB에 `SELECT` 쿼리를 수행하여 해당 고객의 예약 정보를 조회.
    - **조건 분기:**
        - 조회 결과가 존재하면: 예약 리스트 및 저장된 취소율 통계 데이터를 JSON 배열로 패킹하여 `status: "success"`와 함께 리턴.
        - 조회 결과가 없으면: `status: "empty"`, `message: "일치하는 고객이 없습니다."` 형태의 JSON 리턴.
- **데이터베이스 변화 (Oracle 11g):**
    - 데이터의 변경 없이 `SELECT` 쿼리만 수행.

### 기능 ID: F-03-02

- **기능명:** 검색된 내역 내 예약 취소 트리거 및 취소율 상승 반영
- **화면 및 UI 구성:**
    - F-03-01 결과로 출력된 예약 내역 리스트의 각 행(Row) 우측에 **[예약 취소]** 버튼 배치.
- **프론트엔드 액션 (Fetch API):**
    - 특정 항목의 [예약 취소] 버튼 클릭 시, `Confirm` 창으로 재확인.
    - 확인 시, 해당 예약의 ID(`reserve_id`)를 `Fetch API`를 통해 `/api/cancel` 엔드포인트로 **POST** 요청.
    - 취소 성공 응답 수신 시, 화면을 새로고침하지 않고 **해당 예약 행의 상태를 '취소'로 변경**하고, 통계 시각화 컴포넌트의 **누적 취소율 수치를 실시간으로 상승시켜 재렌더링**.
- **백엔드(Flask) 로직:**
    - `/api/cancel` (POST) 엔드포인트에서 `reserve_id` 수신.
    - DB에서 해당 예약의 상태를 'CANCELED'로 업데이트.
    - 전체 세션 또는 해당 카테고리의 누적 취소 통계를 재계산하여 갱신된 취소율 데이터를 결과에 포함.
    - 성공 메시지와 업데이트된 시각화용 통계 데이터를 JSON으로 리턴.
- **데이터베이스 변화 (Oracle 11g):**
    - `RESERVATION` 테이블에서 해당 `RESERVE_ID` 행의 `STATUS` 컬럼을 `'CANCELED'`로 `UPDATE`.
