# 📄 [프로젝트 요구사항 분석서] AI 기반 호텔 예약 및 취소율 예측 시스템

## 1. 개요 (Introduction)

- **프로젝트명:** AI 기반 호텔 및 세미나실 실시간 예약 관리 및 취소율 예측 웹 서비스
- **목적:** 일반 고객에게 편리한 호텔 객실 및 세미나실 예약 환경을 제공하고, 고객의 예약 조건(체크 리스트)을 기반으로 실시간 취소율을 예측하여 시각화함으로써 데이터 기반의 자원 관리 역량을 확보한다.
- **주요 특징:** * Flask와 Oracle 11g(Thick 모드)를 연동한 안정적인 백엔드 구축
    - `Fetch API`를 활용한 비동기(Asynchronous) 데이터 통신으로 화면 새로고침 없는 매끄러운 UX 제공
    - 기학습된 머신러닝 모델을 통한 실시간 취소 확률 계산 및 데이터 시각화

## 2. 시스템 환경 및 기술 스택 (System Architecture & Tech Stack)

- **Language:** Python 3.11.9
- **Framework:** Flask (Python Web Framework)
- **Database:** Oracle Database 11g
    - *특이사항:* `oracledb` 라이브러리 v1.4.2 버전을 사용하며, Oracle 11g 환경 호환을 위해 **Thick Mode**로 클라이언트 환경을 구성함.
- **Frontend:** HTML5, CSS3, JavaScript ~~(Modern ES6+)~~
    - *핵심 기술:* 비동기 통신을 위한 **Fetch API** 적극 활용
- **Data & AI:** 취소율 예측을 위한 머신러닝/딥러닝 모델 (Flask 서버 내에 탑재 및 서빙)

## 3. 사용자 및 시스템 요구사항 (User & System Requirements)

### 3.1 대상 사용자 (Target Audience)

- **일반 고객:** 호텔 객실이나 세미나실 이용을 원하는 사용자. 웹사이트에 접속하여 실시간 예약을 진행하고 본인의 예약 상태를 관리(조회/취소)함.

### 3.2 기능적 요구사항 (Functional Requirements)

### [F-01] 회원 및 기본 접근 (접속)

- 고객은 웹사이트에 접속하여 객실 및 세미나실 예약 화면에 접근할 수 있어야 한다.

### [F-02] 예약 및 조건 선택 (핵심 기능)

- 고객은 **호텔 객실** 또는 **세미나실** 중 원하는 자원을 선택하여 예약을 진행할 수 있어야 한다.
- 예약 시, 예측 모델에 필요한 특정 조건(체크 리스트)을 입력하거나 선택해야 한다.
    - *(예: 투숙 기간, 주말 숙박 여부, 추가 요청 사항 등 모델의 피처(Feature)로 들어갈 데이터 항목)*

### [F-03] 실시간 취소율 예측 및 피드백 (AI 연동)

- 고객이 예약을 완료하거나 조건을 변경할 때, 시스템은 백엔드 Flask 서버에 적재된 AI 모델을 호출해야 한다.
- 모델은 고객이 체크한 리스트를 기반으로 **해당 예약의 취소 확률(취소율)을 예측**한다.
- 예측된 결과 및 취소율 데이터는 화면에 시각적으로 레이아웃(차트 또는 인디케이터 형태)되어 사용자나 관리자에게 제공되어야 한다.

### [F-04] 예약 관리 및 상태 변경 (데이터 유기적 연동)

- 고객은 '본인의 예약 관리' 페이지에 접근하여 현재 예약 내역을 조회할 수 있어야 한다.
- 고객이 예약 관리 페이지에서 **'예약 취소' 버튼을 누르면, 해당 데이터가 시스템에 반영**되어야 한다.
- 실제 취소 이벤트가 발생함에 따라, 전체 혹은 해당 그룹의 통계 데이터 상 **누적 취소율이 실시간 또는 주기적으로 상승 반영**되어야 한다.

### 3.3 비기능적 요구사항 (Non-Functional Requirements)

### [NF-01] 성능 및 사용자 경험 (UX)

- 예약, 취소, 취소율 데이터 갱신 등 주요 데이터 요청 처리는 사용자의 화면 흐름을 끊지 않도록 **Fetch API를 통한 비동기(Asynchronous) 방식으로 처리**되어야 한다.

### [NF-02] 데이터 호환성 및 안정성

- Oracle 11g 및 `oracledb (v1.4.2) Thick mode` 환경에서 커넥션 풀링 및 쿼리 수행이 안정적으로 이루어져야 한다.

## 4. 핵심 시나리오 (Core User Flow)

1. **접속 및 선택:** 고객이 사이트 진입 후 호텔 객실 또는 세미나실 선택
2. **조건 입력:** 예약 세부 조건(체크 리스트)을 입력 후 예약 요청
3. **예측 및 저장:** Flask가 Oracle DB에 예약을 저장함과 동시에 AI 모델을 돌려 취소율을 계산하고 화면에 비동기(Fetch)로 시각화 결과 전송
4. **마이페이지:** 고객이 예약 관리 메뉴 진입
5. **취소 트리거:** 고객이 취소 요청 시, Fetch API를 통해 DB 상태가 '취소'로 변경되고, 모델/시각화 파트의 누적 취소율 지표가 즉시 상승함.
